7. tilt       - radar mounting and pointing
8. rotation  - radar mounting and pointing

9. range delay - radar hardware

The matrix xmat_vsurf and vector vect_vsurf are filled in kdzsurf_kvsurf_ge_1 function.
The matrix and vector linear equations are then solved using chol_inv function. 
The sine and cosine of various variables are calculated and stored in a common block converted to a dictionary in the azel.py function.
There are 12 variables/corrections to solve for using the xmat_vsurf and vect_vsurf matrix and vector.  The variables are referenced using the variable_indexes.py structure, which is an include file / global to import as needed. 
The DTM is a Detailed Terrain Map, which is not used based on the sample DATA_cns.txt input parameter files.  If the "use DTM" flag is not set in the input parameter file, the internal alt_dtm matrix is set to a constant z?? which seems to imply a flat earth.
var[i] is used to store intermediate values, which are then used to complete the non-diagonal elements of the xmat matrix.
